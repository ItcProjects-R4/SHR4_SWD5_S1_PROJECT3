# التوثيق الشامل لمشروع إطمن (Etmen Project Documentation)

منصة **إطمن (Etmen)** هي نظام رقمي متكامل لإدارة الرعاية الصحية الطارئة، القدرات الاستيعابية للمنشآت الطبية، والتعامل الذكي التزامني مع الأزمات الصحية والأوبئة. تم تطوير المشروع ليكون نموذجاً تطبيقياً متميزاً يتماشى مع رؤية مصر 2030 للتحول الرقمي لقطاع الرعاية الصحية بالجمهورية.

## 🌟 هوية المشروع وفلسفته

* **الاسم:** **إطمن (Etmen)** - رسالة طمأنينة مباشرة للمريض وعائلته أثناء مواجهة الطوارئ الصحية والذعر.
* **شعار المنصة:** *"التكنولوجيا في خدمة الحياة.. رعاية صحية أسرع، حياة أكثر أماناً"*
* **الفلسفة الأساسية:**  
  التحول بالاستجابة للطوارئ من الأسلوب العشوائي التقليدي (البحث اليدوي الهاتفي عن سرير شاغر أو مستشفى يقبل الحالة) إلى أسلوب مؤتمت لحظي يربط المريض بأقرب سيارة إسعاف وأقرب مستشفى مناسب متاح جغرافياً ويمتلك الموارد اللازمة (أسرة رعاية مركزة ICU، أطباء مناوبين) بنقرة زر واحدة.

## 🏗️ البنية المعمارية للنظام (Clean Architecture)

يعتمد الحل البرمجي لـ **إطمن** على معمارية **الهندسة النظيفة (Clean Architecture)** لضمان فصل الاهتمامات (Separation of Concerns)، استقلالية النظام عن قواعد البيانات والمكتبات الخارجية، وسهولة إجراء الاختبارات البرمجية وتحديث الأكواد دون إحداث أخطاء جانبية.

```
                  ┌─────────────────────────────┐
                  │          Presentation       │
                  │           (Etmen_PL)        │
                  └──────────────┬──────────────┘
                                 │
                                 ▼
                  ┌─────────────────────────────┐
                  │        Business Logic       │
                  │          (Etmen_BLL)        │
                  └──────────────┬──────────────┘
                                 │
                                 ▼
                  ┌─────────────────────────────┐
                  │          Data Access        │
                  │          (Etmen_DAL)        │
                  └──────────────┬──────────────┘
                                 │
                                 ▼
                  ┌─────────────────────────────┐
                  │          Core Domain        │
                  │        (Etmen_Domain)       │
                  └─────────────────────────────┘
```

## ⚙️ تفاصيل الطبقات والمشاريع الفرعية

### 1. طبقة النطاق الأساسية: `Etmen_Domain`
هي قلب المعمارية وتخلو تماماً من أي مكتبات خارجية أو علاقات بـ Frameworks. تحتوي على:
* **Entities**: الكيانات الأساسية لقاعدة البيانات (مثل `PatientProfile` و `DoctorProfile` و `EmergencyRequest` و `CrisisConfiguration` و `MedicalRecord`).
* **Enums**: الثوابت الطبية والتشغيلية الموحدة في النظام (مثل `RiskLevel` و `EmergencyRequestStatus` و `AppointmentStatus` و `StaffRoleType`).

### 2. طبقة الوصول للبيانات: `Etmen_DAL`
مسؤولة عن التعامل المباشر مع قاعدة البيانات وتنفيذ العمليات المستمرة:
* **DbContext (`EtmenDbContext`)**: سياق الاتصال بقاعدة البيانات. يرث من `IdentityDbContext<ApplicationUser>` لدعم جداول الهوية، وتكوين الجداول والعلاقات المعقدة عبر **Fluent API** في مجلد `Configurations`.
* **Repository Pattern & Unit of Work**:
  * `GenericRepository<T>`: لتسهيل عمليات الإدخال والتعديل والحذف والقراءة العامة.
  * `Unit of Work (`UnitOfWork`)`: لجمع كافة التغييرات وحفظها في قاعدة البيانات في عملية واحدة تضمن سلامة وتكامل البيانات (Transactions).
* **Seeding**: تهيئة النظام بالبيانات الأساسية وتوليد الحسابات الوظيفية وأوزان الأعراض والأزمات الافتراضية عبر `DataSeeder`.

### 3. طبقة منطق الأعمال والخدمات: `Etmen_BLL`
تضم كافة القواعد الطبية، خوارزميات الفرز، والربط بالـ APIs الخارجية:
* **Services**: تطبيق المنطق الطبي والتشغيلي (مثل `CrisisRiskEngineService` و `LabService` و `AIChatService` و `PdfReportService`).
* **DTOs**: كائنات نقل البيانات لتقليل حجم البيانات المتبادلة وعزل الكيانات الحقيقية عن طبقة العرض.
* **Mapping**: تحويل الكيانات لـ DTOs تلقائياً في الذاكرة عبر أداء برمجي عالٍ بالاعتماد على مكتبة **Mapster**.

### 4. طبقة العرض والواجهات: `Etmen_PL`
بوابة تفاعل المستخدمين مع النظام:
* **Controllers**: 29 متحكماً برمجياً لتنظيم ومعالجة طلبات واجهات الويب.
* **Razor Views**: واجهات ويب تفاعلية مبنية بالكامل باستخدام Bootstrap 5 و Vanilla CSS.
* **SignalR Hubs**: قنوات لبث التحديثات الفورية للطوابير وحركات الإسعاف والمحادثات الطبية.
* **Filters**: مثل `DoctorOnboardingFilter` و `PatientProfileFilter` لإلزام المستخدمين باستكمال بياناتهم لسلامة العمل التشغيلي.

## 🗺️ المنظومة الجغرافية والذكاء المكاني (GIS Intelligence)

تتميز منصة **إطمن** بالدمج العميق للخرائط التفاعلية والـ GIS على الويب عبر أربع منظومات جغرافية متميزة:

1. **خريطة تتبع الطوارئ اللحظية (Ambulance Tracking Map):**  
   عند تحريك سيارة الإسعاف لنقل مريض مستغيث، تعرض شاشة استقبال المستشفى تتبعاً حياً لموقع سيارة الإسعاف، موقع المريض، وموقع المستشفى الجغرافي لحساب الزمن المتوقع للوصول (ETA) وتهيئة المسعفين للاستقبال.
2. **الخريطة الحرارية للأوبئة (Crisis Outbreak Heatmap):**  
   تقوم بتجميع المواقع الجغرافية للمرضى الذين تم تشخيصهم بحالات حرجة أو عالية الخطورة، وترسم نطاقات حمراء (Hotspots) فوق بؤر تفشي العدوى لتنبيه وزارة الصحة لفرض حجر طبي أو توجيه دعم طبي مكثف.
3. **خريطة أقرب مقدم خدمة (Nearby Providers Map):**  
   تعتمد على تحديد موقع المريض تلقائياً بالـ GPS ومسح الدائرة المحيطة بقطر 10 إلى 50 كم لعرض كافة المستشفيات والعيادات المتاحة، وكمية الأسرة الشاغرة (Live Capacity) في الوقت الفعلي مع زر حجز فوري للموعد.
4. **الخريطة العامة للشبكة الطبية (Telemetry Network Map):**  
   لوحة قيادية مركزية لمديري النظام (Admin Map) تمنح نظرة بانورامية على توزيع المستشفيات النشطة، العيادات، وسيارات الإسعاف العاملة في البلد.

## 🤖 مميزات الذكاء الاصطناعي ومعالجة البيانات

* **مساعد الاستشارات الطبية الذكي (Gemini AI Chatbot):**  
  تم توفير شات بوت تفاعلي متكامل مع محرك جوجل للذكاء الاصطناعي `gemini-1.5-flash` ليقدم إرشادات وقائية أولية للمرضى، ويجيب عن استفساراتهم الطبية العامة بشكل آمن ومبني على الـ Prompt Engineering الصحي السليم.
* **محرك استخلاص التحاليل بالـ OCR (AI-OCR Lab Analyzer):**  
  تتيح المنصة للمريض رفع صورة للتحاليل الطبية (مثل تحليل CBC أو وظائف الكلى) ليقوم النظام تلقائياً عبر تكامله مع مكتبات الـ OCR بقراءة النصوص والأرقام، وتصفيتها بمطابقة النطاق الطبيعي، واستخلاص النسب الشاذة وعرضها باللون الأحمر فوراً للطبيب كإنذار مبكر للتدهور الصحي.

## 🔄 سيناريوهات تدفق العمل التزامني (Workflows)

```
┌────────────────────────────────────────────────────────┐
│ 🤕 المريض: يطلق استغاثة طارئة (Panic Button)           │
└──────────────────────────┬─────────────────────────────┘
                           │
                           ▼
┌────────────────────────────────────────────────────────┐
│ 🧠 محرك الفرز: يحسب الخطورة ويحدد الموقع الجغرافي      │
└──────────────────────────┬─────────────────────────────┘
                           │
                           ▼
┌────────────────────────────────────────────────────────┐
│ 🏨 المستشفى: يستلم البلاغ في طابور الطوارئ الفوري      │
└──────────────────────────┬─────────────────────────────┘
                           │
                           ├─────────────────────────────┐
                           ▼                             ▼
┌─────────────────────────────────────────┐  ┌──────────────────────────────────────────┐
│ 🚑 الإسعاف: تنطلق ويتم تتبعها حياً      │  │ 👨‍⚕️ الطبيب: يتلقى نداء الذعر في صندوقه    │
└─────────────────────────────────────────┘  └──────────────────────────────────────────┘
```

### 1. تدفق الطوارئ والاستغاثة (Emergency Dispatch Flow)
1. يضغط المريض على زر الاستغاثة السريع (Panic Button) من لوحة تحكمه.
2. يقوم محرك المخاطر `CrisisRiskEngineService` بالتقاط موقع المريض وحساب درجة خطورة حالته الطبية بناءً على علاماته الحيوية.
3. يتم إرسال طلب استغاثة مزود بموقع المريض الجغرافي وتقريره الطبي إلى شاشة طابور الطوارئ الحية بالمستشفى الأقرب (`HospitalQueueController`).
4. يتم تنبيه طبيب الطوارئ المناوب في صندوق استقبال الذعر الخاص به (`DoctorPanicInboxController`) مع إرسال إشعار فوري بالبريد لعائلته.
5. تبدأ سيارة الإسعاف التحرك، ويتم تتبع إحداثيات حركتها حياً عبر الخريطة من قبل كل من المستشفى والمريض.

### 2. تدفق إطلاق وضع الأزمات والأوبئة (Epidemic Declaration Flow)
1. عند زيادة معدلات الإصابة، يقوم مدير النظام بتهيئة وباء جديد في لوحة تحكم الأزمات (`AdminCrisisController`).
2. يربط المدير الأعراض المرافقة (مثل ضيق التنفس، السعال الجاف، فقدان الشم) ويعين لها أوزاناً رقمية من (0.00 إلى 1.00).
3. يتم تفعيل **وضع الأزمة (Crisis Mode)** تلقائياً في النظام.
4. يتكيف محرك التقييم الذاتي للمرضى لتتصدر الأعراض الجديدة الأسئلة المطروحة، ويتم تغيير طريقة حساب درجة المخاطر فوراً في قاعدة البيانات ودون توقف النظام.

## 🔌 قنوات الاتصال بالوقت الفعلي (SignalR Hubs)

يعتمد النظام على **ASP.NET Core SignalR** لتأمين بث البيانات وتدفقها الفوري بالخلفية دون الحاجة لإعادة تحميل الصفحات:
* **`ChatHub`:** لبث رسائل المحادثات والاستشارات الفورية الفورية بين المريض وطبيبه المعالج.
* **`QueueHub`:** لتحديث طوابير فرز الطوارئ (Emergency Triaging Queues) في شاشات المستشفيات فور إطلاق أي استغاثة.
* **`EmergencyHub`:** لبث الإحداثيات الجغرافية الحية لسيارات الإسعاف أثناء تحركها وتحديث موقعها على الخريطة كل ثوانٍ.

## 🛡️ الهوية والأمان والصلاحيات

* **إدارة الحسابات:** يعتمد النظام بالكامل على **ASP.NET Core Identity** لتأمين الحسابات، تشفير كلمات المرور، وإدارة الجلسات (Sessions).
* **تأكيد البريد والتحقق:** لا يتم تفعيل حساب المريض أو الطبيب بشكل تشغيلي إلا بعد تأكيد البريد الإلكتروني برابط تحقق وتوكن مشفر فريد.
* **فلاتر التهيئة (Onboarding Filters):**
  * `DoctorOnboardingFilter`: يمنع الطبيب من تصفح لوحته المهنية أو استقبال حجوزات حتى يقوم باستكمال بيانات تخصصه الدقيق، رقم ترخيصه الطبي وموقع عيادته جغرافياً.
  * `PatientProfileFilter`: يمنع المريض من استخدام خدمات الطوارئ أو الاستشارة حتى يستكمل بياناته الطبية الحرجة كفصيلة الدم والأمراض المزمنة لتفادي إطلاق استغاثات مجهولة البيانات.

## ⚡ الخدمات الخلفية المجدولة (Background Services)

يتم تشغيل المهام الثقيلة والمجدولة تلقائياً في الخلفية باستخدام **Hosted Services (IHostedService)**:
* **`AppointmentReminderHostedService`:** خدمة تعمل تلقائياً في الخلفية كل 30 دقيقة للتحقق من المواعيد الطبية المحجوزة، وتقوم بإرسال رسائل بريد إلكتروني تذكيرية للمرضى قبل موعد الحجز بـ 24 ساعة ثم قبل الحجز بساعتين.
* **`QueuedHostedService`:** لتأمين تشغيل المهام الخلفية الكبيرة مثل تصدير التقارير المعقدة وملفات الـ PDF والتحاليل دون حدوث بطء في واجهات المستخدمين.

## 🛠️ التقنيات والمكتبات المستخدمة

* **إطار العمل البرمجي:** .NET 10.0 (ASP.NET Core MVC).
* **قاعدة البيانات:** Microsoft SQL Server.
* **مستعرض وعلاقات قاعدة البيانات (ORM):** Entity Framework Core 10.
* **نقل البيانات والتطابق:** Mapster (بديل عالي الأداء لـ AutoMapper).
* **محرك المحادثات اللحظية:** SignalR.
* **توليد التقارير:** مولد مستندات PDF مدمج مع خدمات إرسال البريد الإلكتروني SMTP.
* **واجهات المستخدم:** HTML5, CSS3, Javascript, Bootstrap 5.
* **تكامل الذكاء الاصطناعي:** Google Gemini API (`gemini-1.5-flash`).
* **الخرائط التفاعلية:** LeafletJS / Mapbox لتقديم رسومات الخرائط وتحديد إحداثيات الـ GIS.
